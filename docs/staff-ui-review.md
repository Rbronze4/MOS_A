# スタッフ側画面 UX/機能レビュー

> 作成日: 2026-07-01
> 対象: スタッフ（店員）側の全画面
> 目的: チーム開発メンバー向けに、UX上の問題点・不足機能を共有し、改善タスクの優先付けを行う

## 対象画面

ログイン / ホーム / 注文一覧 / 顧客詳細 / 注文詳細 / 商品管理 / QR発行 / スタッフ注文（卓入力・メニュー）

## 前提

- 本システムは **DB未接続**。データはPHPにハードコード、状態はフロント（JSメモリ + sessionStorage）のみで保持。
- そのため下記の指摘には「DB化すれば自然に解決するもの」と「今のフロント実装内でも直すべきもの」が混在する。各項目に区別を記載。

---

## 🔴 バグ・不整合（優先度：高）

### 1. QR発行の店舗ID保存が実装と噛み合っていない
- **現象**: `public/assets/js/staff/dashboard/qr.js` が `localStorage.getItem('mos_current_store_id')` を読むが、ログイン処理（`dashboard.js`）に**保存している箇所がない**。常にフェイルセーフの `'MB'` になる。しかもログイン画面の店舗コードに `MB` は存在しない（緑橋本店は `MH`）。
- **影響**: QRのURLが実在しない店舗IDになる。
- **区分**: フロント実装バグ（要修正）。

### 2. QR発行の「枚数」入力が使われていない
- **現象**: `qr_issue.php` に枚数欄があるが、`dashboard.js` は人数しか見ておらず、**枚数は完全に無視**。
- **影響**: 1回1枚しか発行できず、複数QRを一度に出せない。
- **区分**: 機能未実装（要対応）。

### 3. スタッフ注文メニューの飲み放題プラン判定が客側とズレる
- **現象**: `staff_order_menu.php` は `in_array($plan, ['standard','premium'])` の完全一致。
- **影響**: 実装計画書にある**4コース化（`standard_120` 等）で壊れる**既知の箇所。また客側との0円判定ロジックが二重管理になっている。
- **区分**: 将来の4コース化と併せて要リファクタ（`free_drink` フラグ判定へ）。

### 4. スタッフ注文のカテゴリが客側・商品管理と食い違う
- **現象**:
  - スタッフ注文メニュー: `ドリンク / ご飯もの / 串 / 一品 / 揚げ物 / 限定`
  - 客側: `ドリンク / 串 / 一品 / 揚げ物 / ご飯もの / 期間限定 / 店舗限定`
- **影響**: 「限定」だけ／「期間限定・店舗限定」で分かれる等バラバラ。商品管理は客側に揃え済みなので、スタッフ注文だけ取り残されている。
- **区分**: 要統一（軽微）。

### 5. メニュー画像パスの大文字・小文字不一致
- **現象**: `Chicken_thigh.png` / `Chicken_skin.png` と参照しているが、実ファイルは `chicken_thigh.png`（小文字）。
- **影響**: **Windowsでは表示されるがLinux本番では画像切れ**になる。客側 `CustomerController.php` も同じ問題。
- **区分**: 要修正（本番で顕在化する潜在バグ）。

## 🟡 UX上の問題（優先度：中）

### 6. スタッフ注文「決定」後の完了フィードバックが弱い
- **現象**: `order-menu.js` は `alert('注文を受け付けました')` のみで、**元の画面に戻らない**。
- **影響**: 注文したのにメニュー画面に留まり、次の操作が不明瞭。注文一覧や注文詳細へ自動で戻したい。
- **区分**: UX改善（軽微）。

### 7. パスワードが実質ノーチェックでログインできる
- **現象**: `dashboard.js` は「空でなければOK」。何を入れても通る。
- **影響**: デモ段階なら可だが、**誤操作で別店舗としてログインしても気づけない**。
- **区分**: DB/認証実装時に対応。

### 8. 注文一覧に「席」でのまとまり・絞り込みがない
- **現象**: 全注文がフラットに並ぶだけ。
- **影響**: 実運用では**卓ごとにまとめて見たい／特定卓で絞りたい**はず。件数が増えると提供作業がしづらい。
- **区分**: UX改善（中規模）。

### 9. 「操作の取り消し」導線が弱い
- **現象**: 一覧の提供操作（1つ提供など）は即時反映で、確認も取り消しもなし。「提供取消」はあるが、一括キャンセルだけ confirm あり・個別操作は confirm なし。
- **影響**: 操作基準が不統一で、誤操作のリカバリがしにくい。
- **区分**: UX改善（軽微）。

### 10. ログアウトに確認がない
- **現象**: ホームの「ログアウト」やサイドメニューのログアウトは**即座にログイン画面へ**。
- **影響**: 誤タップで作業中の状態が飛ぶ。
- **区分**: UX改善（軽微）。

### 11. QR発行後、発行した顧客番号が記録・一覧化されない
- **現象**: モーダルで番号を出すだけで、**どの卓に何番を割り当てたか後から確認できない**。
- **影響**: 顧客詳細一覧と紐づいておらず、運用で追跡できない。
- **区分**: DB化と併せて対応。

## 🟢 不足していそうな機能（あると良い）

### 12. 会計（清算）機能が見当たらない
- README の前提仕様に「会計済みにする（billStatus=2）」とあるが、**スタッフ画面に会計・清算の導線がない**。注文一覧は提供/キャンセルのみ。居酒屋オーダーとしては会計が要のはず。

### 13. 合計金額・卓ごとの売上表示がない
- 注文詳細に個数はあるが**金額列・卓の合計がない**。会計前に「この卓いくら？」が即座に分からない。

### 14. 新規注文の通知・リアルタイム更新がない
- 客側で注文が入ってもスタッフ画面は自動更新されない（DB未接続のため）。実運用では**新着注文のバッジ／音／自動リロード**が欲しい。

### 15. 商品管理に「品切れ（提供停止）」トグルがない
- 在庫管理は不要にしたが、**「今日は売り切れ」で一時的に客側メニューから隠す**機能はニーズがありそう。

### 16. 空状態（データ0件）の表示がない
- 注文一覧・注文詳細・商品管理で**該当0件のとき何も出ない**（空の表だけ）。「注文はありません」等の空メッセージが欲しい。

---

## おすすめ着手順（提案）

まず**実害のあるバグ系**から潰すのが良い。工数も小さいものが多い。

1. **#5 画像パスの大文字小文字**（本番で画像切れ・全画面共通、修正は軽い）
2. **#4 スタッフ注文カテゴリを客側に統一**（軽い）
3. **#1・#2 QR発行の店舗ID保存＋枚数対応**（機能が実質壊れている）
4. **#6 注文確定後に元画面へ戻す＋トースト**（体感UXが大きく改善、軽い）
5. その後 **#12 会計機能** など大きめの機能へ

## 補足：本PRに含む修正

このレビューと同じPRで、以下の改善を先行して実施済み（商品管理・注文編集まわり）。

- 商品管理: 在庫欄（一覧・追加/編集フォーム）を削除
- 商品管理: 追加/編集フォームのカテゴリを客側の7種に統一（ご飯もの・期間限定・店舗限定を追加）
- 商品管理: 値段（税抜）入力で消費税10%を自動計算し、値段（税込）を自動表示（税込を保存）
- 注文編集モーダル: キャンセルボタンを追加（枠外クリックでも閉じられる）
