# MOS開発でAIを使うときのルールとプロンプト

MOSを多人数で開発する場合、AIに自由にコードを書かせると、既存の画面・JavaScript・DB・API仕様を壊してしまう可能性があります。

HTMLの `id`、CSSの `class`、JavaScriptの関数名、PHPのルート、DBカラム名、API仕様がつながっています。
そのため、AIが勝手に名前を変更すると、画面は表示されてもボタンが動かない、DBに登録できない、API連携できないといった問題が起きます。

そのため、AIに依頼するときは、毎回以下のルールを守ってください。

---

## AIに依頼するときの基本ルール

### 1. 1回の依頼で1機能だけ頼む

悪い例：

```text
注文一覧を直して、スタッフ注文も追加して、ログアウトの404も直して、客側のタブ矢印も追加してください。
```

良い例：

```text
今回はログアウトボタン押下時の404エラーだけを修正してください。
```

1回の依頼で複数の機能をまとめて頼むと、AIが関係ない部分まで変更してしまう可能性があります。

---

### 2. 変更してよいファイルを明記する

AIが関係ないファイルまで勝手に変更しないように、必ず変更してよいファイルを書きます。
また、変更すべきファイルをAIに特定させて、ファイルを変更してよいか確認してください。

例：

```text
【変更してよいファイル】
- src/Views/staff/order_list.php
- public/assets/css/staff.css
- public/assets/js/staff-order.js
```

---

### 3. 現在のコードを必ず貼る

AIに想像でコードを書かせると、存在しないファイル名・関数名・DBカラム名を使う可能性があります。

そのため、必ず現在のコードを貼ってから依頼してください。

```text
以下が現在のコードです。
このコードをベースに修正してください。
省略せず完全版で返してください。
```

---

### 4. 既存のID・class・関数名・ルート名を勝手に変えさせない

HTMLの `id` や CSSの `class` は、JavaScriptやCSSとつながっています。

例えば、既存コードで以下のように使っている場合、

```html
<button id="cartButton">カート</button>
```

AIが勝手に以下のように変更すると、

```html
<button id="cart-btn">カート</button>
```

JavaScript側で `cartButton` を探している処理が動かなくなる可能性があります。

そのため、以下の指示を必ず入れてください。

```text
既存のid、class、関数名、ルート名を勝手に変更しないでください。
```

---

### 5. DB構造を勝手に変更させない

DB変更は影響範囲が大きいため、AIに勝手に追加・変更させないでください。

```text
DB構造を勝手に変更しないでください。
DB変更が必要な場合は、先に理由を説明してください。
```

---

### 6. コードは省略なしの完全版で出してもらう

一部だけ出されると、どこに貼ればよいか分かりにくく、ミスが起きやすいです。

```text
コードは省略しないでください。
「既存部分はそのまま」などの省略表現は使わず、コピペで差し替えられる完全版を出してください。
```

---

### 7. コメントは多めに書いてもらう

後から別のメンバーが読んでも処理の流れが分かるように、コメントを多めに書いてもらいます。

ただし、当たり前すぎるコメントではなく、**なぜその処理が必要なのか** が分かるコメントを優先してください。

悪いコメント例：

```php
// if文です
if ($status === 2) {
}
```

良いコメント例：

```php
// 会計済みの注文は二重会計を防ぐため、再度清算処理に進ませない
if ($status === 2) {
}
```

---

### 8. 最後に動作確認手順も出してもらう

コードを出して終わりではなく、何を確認すればよいかもAIに出してもらってください。

例：

```text
【動作確認】
1. 注文一覧画面を開く
2. 「1つ提供」を押す
3. 提供数が1増えることを確認する
4. 「1つ減らす」を押す
5. 提供数が1減ることを確認する
6. 提供数が0未満にならないことを確認する
```

# AIに貼るプロンプトテンプレート

以下をコピーして、必要な部分を書き換えて使ってください。

```text
あなたはMOS開発の補助AIです。
以下の既存仕様と開発ルールを必ず守ってください。

【開発環境】
- PHP
- JavaScript
- HTML/CSS
- MySQL
- XAMPP
- MVC構成

【MOSの前提仕様】
- 居酒屋向けのモバイルオーダーシステムです
- 客側画面、店員側画面、レジ側画面があります
- 注文一覧、提供済み一覧、キャンセル済み一覧があります
- APIは POST /api/orders を使用します
- method は getOrders または updateStatus です
- customerId は7桁です
- storeId は大文字アルファベット2文字です
- 会計済みにする場合は billStatus=2 です
- getOrders は注文取得のみで、状態更新はしません
- updateStatus で注文状態を更新します

【今回の目的】
ここに今回実現したい内容を書いてください。

例：
スタッフ側の注文一覧画面で、「1つ提供」「全て提供」「1つ減らす」の3つのボタンを1行で表示し、それぞれ正しく提供数を変更できるようにしたいです。

【現在の問題】
ここに現在困っていることを書いてください。

例：
現在は操作ボタンが2段になっていて見た目が悪いです。
また、「1つ減らす」を押したときに注文個数まで減ってしまう可能性があります。

【変更してよいファイル】
以下のファイルだけを変更してください。

- ここに変更してよいファイル名を書く
- 例：src/Views/staff/order_list.php
- 例：public/assets/css/staff.css
- 例：public/assets/js/staff-order.js

【変更してはいけないこと】
- 指定したファイル以外は変更しないでください
- 既存のid、class、関数名、ルート名を勝手に変更しないでください
- DB構造を勝手に変更しないでください
- 既存機能を削除しないでください
- 指示していない画面に影響を出さないでください
- API仕様を勝手に変更しないでください
- 既存コメントはできるだけ残してください
- 処理を大きく作り直す場合は、先に理由を説明してください

【コメント方針】
- 初学者でも処理の流れが分かるように、コメントを多めに書いてください
- コメントは日本語で書いてください
- 特に、条件分岐・ループ・DB処理・API通信・画面更新処理にはコメントを付けてください
- 「何をしているか」だけでなく、「なぜその処理が必要なのか」が分かるコメントを優先してください
- MOS独自の仕様に関係する処理には必ずコメントを入れてください
- ただし、1行ごとに当たり前の説明をするコメントは避けてください

悪いコメント例：
「if文です」
「変数に代入します」

良いコメント例：
「会計済みの注文は二重会計を防ぐため、再度清算処理に進ませない」
「提供数が注文数を超えないように、最大値を注文数に制限する」

【現在のコード】
以下に現在のコードを貼ります。
このコードをベースに修正してください。
省略せず、差し替え可能な完全版で返してください。

ここに対象ファイルのコードを貼る。

【期待する動作】
以下の動作になるようにしてください。

1. ここに期待する動作を書く
2. ここに期待する動作を書く
3. ここに期待する動作を書く

例：
1. 「1つ提供」を押すと、提供数が1増える
2. 「全て提供」を押すと、提供数が注文個数と同じになる
3. 「1つ減らす」を押すと、提供数だけが1減る
4. 提供数は0未満にならない
5. 注文個数は変更されない

【出力してほしいもの】
以下の順番で出力してください。

1. 修正するファイル一覧
2. 修正内容の概要
3. 各ファイルの完全版コード
4. 変更したポイントの説明
5. 動作確認手順

【注意】
コードは省略しないでください。
「既存部分はそのまま」などの省略表現は使わず、コピペで差し替えられる完全版を出してください。
```

## 使い方の例

例えば、注文一覧画面を修正したい場合は、以下のように書き換えます。

```text
【今回の目的】
スタッフ側の注文一覧画面で、「1つ提供」「全て提供」「1つ減らす」の3つの操作ボタンを1行で表示したいです。

【現在の問題】
現在は操作ボタンが2段になっていて見た目が悪いです。
また、「1つ減らす」では注文個数ではなく、提供数だけを減らしたいです。

【変更してよいファイル】
- src/Views/staff/order_list.php
- public/assets/css/staff.css
- public/assets/js/staff-order.js

【期待する動作】
1. 「1つ提供」を押すと提供数が1増える
2. 「全て提供」を押すと提供数が注文個数と同じになる
3. 「1つ減らす」を押すと提供数が1減る
4. 提供数は0未満にならない
5. 注文個数は変更されない
6. 3つの操作ボタンは1行で表示される
```

# AIにレビューさせる場合のプロンプト

AIが出したコードをレビューさせたい場合は、別のチャットで以下のように聞くとよいです。

```text
以下のコードをレビューしてください。

観点：
- 既存仕様を壊していないか
- idやclass名の変更でJSが動かなくならないか
- DBカラム名に誤りがないか
- API仕様と矛盾していないか
- SQLインジェクションの危険がないか
- XSSの危険がないか
- 例外処理が不足していないか
- 多人数開発で競合しやすい変更がないか

修正コードはいきなり出さず、まず問題点だけ挙げてください。
```

---

# チーム内でのおすすめ運用

AIを使うときは、以下の流れにしてください。

```text
1. 修正したい内容を1つに絞る
2. 変更してよいファイルを決める
3. 現在のコードをAIに貼る
4. AIに完全版コードを出してもらう
5. 出力されたコードをそのまま信用せず、人間が確認する
6. 動作確認手順どおりにテストする
7. 問題なければGitで共有する
```

# DBのインポート方法

MOSを動かす前に、必要に応じてデータベースを作成し、SQLファイルをインポートしてください。

## 手順

1. XAMPPを起動する
2. XAMPPの画面から **Apache** と **MySQL** を起動する
3. MySQLの **Admin** をクリックして、phpMyAdminを開く
4. `mos_a_system` データベースをまだ作成していない場合は、以下の内容で新規作成する

   * データベース名：`mos_a_system`
   * 文字コード：`utf8mb4_unicode_ci`
5. 作成した `mos_a_system` データベースを選択する
6. 画面上部の **インポート** をクリックする
7. インポートするSQLファイルを選択する
8. **インポート実行** をクリックする

## 注意点

* すでに `mos_a_system` データベースが存在する場合は、新規作成は不要です
* インポート先のデータベースを間違えないように、必ず `mos_a_system` を選択してからインポートしてください
* 文字コードは `utf8mb4_unicode_ci` を選択してください
* インポート時にエラーが出た場合は、同じテーブルがすでに存在していないか確認してください
